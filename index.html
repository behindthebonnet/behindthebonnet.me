Step 1

Download Visual Studio Code

Install VS Code ‚Äî your code editor. Use the primary link below (Windows 64-bit user installer). If it doesn‚Äôt work, use the fallback general download page.

Step 2

Install VS Code Extensions

Open VS Code and click the Extensions icon (four squares). Install two extensions: GitHub Copilot and an HTML preview tool.

  1. Open the Extensions panel.
  2. Search for: GitHub Copilot & install.
  3. Search for: HTML Preview (or ‚ÄúLive Preview‚Äù by Microsoft) & install.
  4. If it asks, click Accept or Trust Publisher.
What if I can't find HTML Preview?

Install Live Server or Live Preview instead. They let you see changes in real time.

Step 3

Go to Source Control

Click the Source Control icon (it looks like a branch with a dot). Sign in and install Git if needed.

  1. Click Source Control.
  2. Sign in with GitHub (browser window will open ‚Äî authorize).
  3. If prompted to install Git, accept. If not, install manually from: https://git-scm.com/downloads/win.
  4. Restart VS Code after Git installs (if necessary).
If Git is missing, VS Code can't clone your repository or commit changes. Make sure it‚Äôs installed and available in your system PATH ‚Äî the installer usually handles this automatically.
Step 4

Clone Your GitHub Repository

Now bring your code from GitHub to your computer (this is called cloning).

  1. Press Ctrl + Shift + P (Command Palette).
  2. Type: GitHub: Clone Repository and press Enter.
  3. Select the repository named behindthebonnet or behindthebonnet.me.
  4. Choose a folder on your computer where the files will download.
  5. Open the cloned folder in VS Code if it doesn‚Äôt open automatically.

Step 5

Open the index.html File

This is the main file that controls what your homepage looks like.

  1. In the Explorer sidebar, find index.html.
  2. Click it to open in the editor.
  3. If it‚Äôs empty or minimal, you can paste the content you want or use Copilot to scaffold it.
What if there is no index.html?

Create a new file named index.html in the root of the project (not inside a subfolder) and add starter HTML.

Step 6

Set Up GitHub Copilot

Turn on Copilot so it can help you write code faster (and suggest HTML as you type).

  1. Click the Copilot icon on the top-right (looks like a robot face).
  2. Sign in with your GitHub/Microsoft account (GitHub Education gives you access).
  3. Open the Copilot chat sidebar if needed.
  4. Switch it into Agent Mode (if available) for conversational edits.
  5. In the chat type requests like: 
    Add a big centered heading that says
Welcome to Behind The Bonnet
and a dark background with white text.
Tip: Press Tab to accept suggestions as you type.
Step 7

Start Building Your Website

You can manually edit the HTML or ask Copilot for changes. Try building structure first, then style.

Core Edits

  • Change the page title inside <title>.
  • Add a heading: <h1>Welcome to Behind The Bonnet</h1>.
  • Add a paragraph or button linking to social media.

Copilot Prompts

  • "Add a footer with my name"
  • "Create a dark theme layout with centered content"
  • "Add a button linking to my Instagram @yourhandle"
  • "Add simple responsive styling"
<section class="hero">
  <h1>Welcome to Behind The Bonnet</h1>
  <p>Exploring what‚Äôs under the hood.</p>
  <a href="https://instagram.com/YOUR_HANDLE" target="_blank">Instagram</a>
</section>
Can I add CSS?

Yes. Add a <style> tag inside <head> or create styles.css and link it.

Step 8

(Optional) Preview Your Site

Preview your changes locally without pushing to GitHub yet.

  1. Right-click index.html.
  2. Select Open with Live Preview (or similar) from the extension.
  3. A preview pane or browser tab opens ‚Äî refresh if needed.
If nothing happens: Install Live Server extension and click "Go Live" in the bottom bar.
Step 9

Save & Publish (Commit & Push)

Once you‚Äôre happy, push changes so they appear on https://behindthebonnet.me.

  1. Press Ctrl + S to save all changes.
  2. Open the Source Control panel.
  3. Type a message like: updated homepage.
  4. Click the Commit / ‚úì button (or sparkle AI message icon if using it).
  5. Then click Push (or it may auto-push if configured).
